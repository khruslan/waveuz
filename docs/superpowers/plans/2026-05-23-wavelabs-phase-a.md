# WaveLabs Phase A — Higgsfield Brand Visual Uplift Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace placeholder media and dishonest hero/stats copy with a coherent Higgsfield-generated asset family and CMO-validated vertical-positioning copy, while keeping all existing animation and i18n behavior intact.

**Architecture:** Data-driven. Extend `data/site.ts` (new `serviceIcons` + `industryPresets` slots; `stats` shape gains `evidence`), update `data/translations.generated.json` for all three locales, render service icons and stat evidence in `components/WaveLabsApp.tsx`, add CSS classes for the new elements. Asset generation through Higgsfield MCP happens last and writes files into `public/media/`. Existing GSAP/Lenis animation untouched.

**Tech Stack:** Next.js 16 App Router, React 18, TypeScript strict, Vitest, Playwright, Higgsfield MCP, ffmpeg for transcoding.

**Spec:** `docs/superpowers/specs/2026-05-23-wavelabs-phase-a-higgsfield-assets-design.md`

**Founder data required before launch (not blocking implementation):**
- Real count of production projects (placeholder `12`).
- Whether "Taking 2 projects this quarter" is accurate (placeholder kept until confirmed).
- Real client names + one quantified outcome per Work case.
